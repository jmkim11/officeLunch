## 오늘의 목표

Spring Boot 백엔드 포트폴리오로 보이도록, 추천 MVP의 첫 번째 기능 단위를 문서와 코드 기준으로 고정한다.

오늘은 기능을 넓히지 않고 다음 흐름만 완성 대상으로 삼는다.

`식당 후보 목록 -> 추천 세션 시작 -> 식당 추천 -> 식당 선택`

## 오늘의 1차 완료 기준

최소 완료 기준은 다음이다.

- `Restaurant`가 실제 데이터 모델 역할을 한다.
- 추천 세션이 같은 식당을 중복 추천하지 않는다.
- 후보 식당이 없을 때 명확히 실패한다.
- 후보에 없는 식당을 선택할 수 없다.
- 선택 완료된 세션은 추가 추천을 받을 수 없다.

## 구현 순서

1. 문서 먼저 고정한다.
2. 도메인 패키지와 클래스 이름을 정리한다.
3. `Restaurant` 모델을 만든다.
4. 추천 세션 규칙을 도메인 코드로 구현한다.
5. 도메인 테스트를 붙인다.
6. Spring MVC 컨트롤러와 DTO를 붙여 Spring Boot 프로젝트로 드러나게 만든다.

## 주의할 점

- 처음부터 DB에 의존하지 않는다.
- 추천 로직은 컨트롤러가 아니라 도메인 객체에 둔다.
- Spring Boot 포트폴리오로 보여야 하므로 도메인만 만든 뒤 멈추지 않는다.
- JPA 매핑은 REST API 흐름이 잡힌 뒤 붙인다.
- 외부 API 키나 지도 API 연동은 MVP 이후로 미룬다.

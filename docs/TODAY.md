## 오늘의 1차 완료 기준

최소 완료 기준은 다음이다.

- `Restaurant`가 실제 데이터 모델 역할을 한다.
- 추천 세션이 같은 식당을 중복 추천하지 않는다.
- 후보 식당이 없을 때 명확히 실패한다.
- 추천받지 않은 식당을 선택할 수 없다.
- 선택 완료된 세션은 추가 추천을 받을 수 없다.
- 후보가 소진된 뒤에도 이미 추천받은 식당을 선택할 수 있다.
- 같은 세션의 동시 요청이 중복 추천이나 선택 덮어쓰기를 만들지 않는다.

## 구현 순서

1. 공통 오류 응답을 정의한다.
2. 추천 세션 생성 기능을 구현한다.
3. 다음 식당 추천 기능을 구현한다.
4. 추천 식당 선택 기능을 구현한다.
5. 각 기능에 테스트와 설계 체크리스트를 붙인다.
6. 전체 범위와 문서 상태를 다시 확인한다.

## 주의할 점

- 이번 1차 MVP는 DB에 의존하지 않는다.
- 추천 로직은 컨트롤러가 아니라 도메인 객체에 둔다.
- Spring Boot 포트폴리오로 보여야 하므로 도메인만 만든 뒤 멈추지 않는다.
- JPA와 Flyway는 실제 영속화 요구가 생기는 다음 단계에서 검토한다.
- 외부 API 키나 지도 API 연동은 MVP 이후로 미룬다.

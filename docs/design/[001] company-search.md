## company-search
## 회사 위치 검색

## 1. 문제
- 왜 필요한가?
- 회사 근처의 식당을 검색하고, 추천하기 위해서는 회사의 위치를 파악할 수 있어야 한다.

## 2. 목표
- 이번 개발에서 무엇을 가능하게 할 것인가?
- 회사 위치 검색 결과를 클라이언트에 반환할 수 있도록 한다.

## 3. 범위
- 무엇을 구현하고 무엇을 구현하지 않을 것인가?
- 외부 API 를 호출하는 방법을 구현할 것이다.
- 회사 선택, 주변 식당 조회, 추천 기능은 이 문서에서 구현하지 않는다.

## 4. 처리 흐름
- 요청부터 응답까지 어떻게 처리되는가?
- 1. 사용자가 회사 이름 또는 근처 역을 입력한다.
- 2. 서버는 외부 장소 API 를 호출해 검색 후보를 조회한다.
- 3. 서버는 검색 후보를 표준 응답 형태로 정리한다.
- 4. 서버는 장소명, 주소, 좌표가 포함된 검색 후보 목록을 클라이언트에 반환한다.


## 5. API와 데이터
- 요청값, 응답값, 주요 데이터 구조는 무엇인가?

- 사용자 요청 -> 시스템 -> 외부 API -> 시스템 -> 검색 후보 목록 반환 흐름이다.

- 1. 사용자의 회사 정보를 입력하면, 시스템은 외부 API 를 호출해 해당 회사 위치 정보를 가져온다.
- 2. 시스템은 회사 위치 정보를 내부 표준 응답 형태로 변환한다.
- 3. 시스템은 변환된 검색 후보 목록을 클라이언트에 반환한다.
- 4. 회사 선택과 주변 식당 조회는 별도 Design Doc에서 다룬다.

## 6. 예외 상황
- 어떤 실패가 발생할 수 있는가?
- 입력값 미정으로 다양한 입력값이 들어왔을 때, 반환이 실패할 수 있다.
- 외부 API 응답에 좌표나 주소가 없을 수 있다.


## 7. 테스트
- 완료됐다고 판단할 기준은 무엇인가?
- 요청값으로 검색 후보 목록을 반환하는지
- 응답값이 정규화 형태로 되는지

## 참고 예시: 회사 위치 검색 Design Doc 초안

> 아래 내용은 위 초안을 지우지 않고 참고하기 위한 예시이다.
> 실제 구현 전에 API 제공자, 응답 필드명, 에러 코드 등은 프로젝트 상황에 맞게 조정한다.

## 1. 문제

Office Lunch는 회사 위치 정보를 기반으로 이후 기능에서 주변 식당을 조회하고 추천한다.
따라서 회사명 또는 주소 입력을 실제 좌표 정보로 변환할 수 있어야 한다.

회사 위치 검색 기능이 불안정하면 클라이언트가 잘못된 회사 후보를 보여줄 수 있다.

## 2. 목표

- 사용자가 회사명 또는 주소로 장소 후보를 검색할 수 있다.
- 검색 결과는 클라이언트가 표시할 수 있도록 장소명, 주소, 좌표를 포함한다.
- 서버는 외부 장소 API 장애가 발생해도 일관된 오류 응답을 반환한다.
- 핵심 로직은 외부 API 없이 테스트할 수 있다.

## 3. 범위

### 구현한다

- 회사명 또는 주소 기반 장소 검색 API
- 외부 장소 검색 API 연동을 위한 Adapter
- 검색어 검증
- 검색 결과 정규화
- 결과 없음, 외부 API 실패, timeout 처리
- Controller, Service, Adapter 단위 테스트

### 구현하지 않는다

- 회사 정보를 DB에 영구 저장하는 기능
- 로그인 사용자별 회사 저장 기능
- 검색 결과 중 회사 선택 기능
- 선택된 회사 기준 주변 식당 조회 기능
- 사용자의 현재 위치 기반 검색
- 자체 지도 화면
- 검색 결과 랭킹 고도화

## 4. 처리 흐름

1. 사용자가 회사명 또는 주소를 입력한다.
2. 클라이언트는 서버에 회사 검색 요청을 보낸다.
3. 서버는 검색어를 검증한다.
4. 서버는 `PlaceSearchPort`를 통해 외부 장소 API를 호출한다.
5. 외부 API 응답을 서비스 내부 표준 형식으로 변환한다.
6. 좌표가 없는 결과는 제외한다.
7. 서버는 검색 후보 목록을 클라이언트에 반환한다.
8. 회사 선택과 주변 식당 조회는 별도 Design Doc에서 다룬다.

## 5. API와 데이터

### 회사 검색 API

```http
GET /api/companies/search?query={query}
```

### 요청 파라미터

| 이름 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| `query` | string | Y | 회사명 또는 주소. 앞뒤 공백 제거 후 2자 이상이어야 한다. |

### 정상 응답

```json
{
  "results": [
    {
      "placeId": "kakao:123456",
      "name": "버킷플레이스",
      "roadAddress": "서울특별시 서초구 서초대로74길 4",
      "jibunAddress": "서울특별시 서초구 서초동 1321-11",
      "latitude": 37.4979,
      "longitude": 127.0276,
      "source": "KAKAO"
    }
  ]
}
```

### 검색 결과 없음 응답

검색 결과가 없는 상황은 서버 오류가 아니므로 `200 OK`와 빈 배열을 반환한다.

```json
{
  "results": []
}
```

### 오류 응답

```json
{
  "code": "INVALID_QUERY",
  "message": "검색어는 2자 이상 입력해야 합니다."
}
```

## 6. 설계 방향

외부 API에 직접 의존하지 않도록 Port와 Adapter를 분리한다.

```text
CompanySearchController
-> CompanySearchService
-> PlaceSearchPort
-> KakaoPlaceSearchAdapter
```

### 핵심 인터페이스 예시

```java
public interface PlaceSearchPort {
    List<CompanySearchResult> searchCompanies(String query);
}
```

서비스는 `PlaceSearchPort`만 의존한다.
Kakao, Naver 등 실제 외부 API 구현은 Adapter에서 담당한다.
이 구조를 사용하면 외부 API 없이 Mock Adapter로 테스트할 수 있고, API 제공자를 바꿔도 핵심 로직 변경을 줄일 수 있다.

## 7. 예외 상황

| 상황 | 처리 |
| --- | --- |
| 검색어가 비어 있음 | `400 Bad Request`, `INVALID_QUERY` |
| 검색어가 1자 이하 | `400 Bad Request`, `INVALID_QUERY` |
| 외부 API 검색 결과 없음 | `200 OK`, 빈 배열 |
| 외부 API timeout | `504 Gateway Timeout`, `PLACE_API_TIMEOUT` |
| 외부 API 인증 실패 | `502 Bad Gateway`, `PLACE_API_AUTH_FAILED` |
| 외부 API 응답 형식 변경 | `502 Bad Gateway`, `PLACE_API_INVALID_RESPONSE` |
| 좌표가 없는 검색 결과 | 해당 결과 제외 |

## 8. 보안

- 외부 장소 API Key는 서버 환경변수로 관리한다.
- API Key를 프론트엔드 코드, Git 저장소, 로그에 남기지 않는다.
- 외부 API 요청/응답 전체를 운영 로그에 그대로 남기지 않는다.
- 사용자 검색어는 앞뒤 공백 제거 후 사용하며, 로그에는 필요한 경우 길이 제한을 적용한다.

## 9. Acceptance Criteria

- 사용자가 2자 이상의 회사명 또는 주소를 입력하면 회사 후보 목록을 조회할 수 있다.
- 검색 결과는 장소명, 도로명 주소, 지번 주소, 위도, 경도를 포함한다.
- 이 API는 클라이언트가 회사 후보를 표시하는 데 필요한 장소명, 주소, 좌표를 반환한다.
- 검색 결과가 없으면 오류가 아니라 빈 목록을 반환한다.
- 외부 장소 API가 지연되어도 서버는 무제한 대기하지 않는다.
- 외부 장소 API 장애가 서버 전체 장애로 이어지지 않는다.
- 외부 API 없이 Mock Adapter로 핵심 테스트를 실행할 수 있다.

## 10. 테스트 케이스

### Service 단위 테스트

- 정상 검색 결과를 표준 응답 형식으로 변환한다.
- 검색어 앞뒤 공백을 제거한다.
- 빈 검색어를 거부한다.
- 1자 검색어를 거부한다.
- 좌표가 없는 결과를 제외한다.
- 외부 API 결과가 없으면 빈 목록을 반환한다.

### Controller 테스트

- `GET /api/companies/search?query=버킷플레이스` 요청 시 `200 OK`를 반환한다.
- 검색 결과 응답에 `name`, `roadAddress`, `latitude`, `longitude`가 포함된다.
- 잘못된 검색어 요청 시 `400 Bad Request`를 반환한다.
- 오류 응답은 `code`, `message` 형식을 따른다.

### Adapter 테스트

- 외부 API 정상 응답을 내부 모델로 변환한다.
- 외부 API timeout을 `PLACE_API_TIMEOUT`으로 변환한다.
- 외부 API 인증 실패를 `PLACE_API_AUTH_FAILED`로 변환한다.

## 11. 보류 사항

- 외부 장소 API 제공자는 Kakao, Naver 중 결정이 필요하다.
- 회사 선택과 주변 식당 조회는 별도 Design Doc에서 결정한다.
- 검색 결과 최대 개수는 기본 5개로 시작하고, 실제 사용성에 따라 조정한다.

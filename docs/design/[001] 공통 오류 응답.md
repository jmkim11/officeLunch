# [001] 공통 오류 응답

## 1. 목적

Controller마다 다른 형태로 오류를 반환하지 않도록 HTTP 상태, 오류 코드, 메시지 형식을 통일한다.

## 2. 구현 범위

- 비즈니스 오류 코드 정의
- 비즈니스 커스텀 예외 정의
- `code`, `message`로 구성된 오류 응답 DTO
- `@RestControllerAdvice`를 사용한 공통 예외 처리
- 요청 DTO 검증 실패와 읽을 수 없는 JSON 처리

## 3. 제외 범위

- 오류 로그 저장
- 외부 모니터링 서비스 연동
- 다국어 오류 메시지
- 모든 예상하지 못한 서버 오류의 강제 변환

## 4. 오류 응답

```json
{
  "code": "RECOMMENDATION_SESSION_NOT_FOUND",
  "message": "추천 세션을 찾을 수 없습니다."
}
```

예상 가능한 비즈니스 실패는 `BusinessException`과 `ErrorCode`로 표현한다. 예상하지 못한 프로그래밍 오류는 숨기지 않고 Spring의 기본 500 응답으로 남겨 원인을 확인할 수 있게 한다.

## 5. 오류 코드

| 코드 | HTTP 상태 | 의미 |
| --- | --- | --- |
| `INVALID_REQUEST` | 400 | 요청값 또는 JSON 형식이 올바르지 않음 |
| `RESTAURANT_NOT_RECOMMENDED` | 400 | 추천받지 않은 식당을 선택함 |
| `RECOMMENDATION_CANDIDATE_NOT_FOUND` | 404 | 조건에 맞는 추천 후보가 없음 |
| `RECOMMENDATION_SESSION_NOT_FOUND` | 404 | 세션 ID에 해당하는 세션이 없음 |
| `RECOMMENDATION_EXHAUSTED` | 409 | 추천 후보를 모두 사용함 |
| `RECOMMENDATION_ALREADY_SELECTED` | 409 | 이미 선택을 완료한 세션을 다시 변경함 |

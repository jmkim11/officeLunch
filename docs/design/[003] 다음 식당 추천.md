# [003] 다음 식당 추천

## 1. 목적

첫 추천이 마음에 들지 않은 사용자가 같은 추천 세션에서 중복 없이 다음 식당을 받을 수 있게 한다.

## 2. 구현 범위

- 세션 ID로 기존 추천 세션 조회
- 이전에 추천한 식당을 제외한 다음 식당 반환
- 변경된 추천 기록 저장
- 존재하지 않는 세션과 후보 소진 처리

## 3. 제외 범위

- 카테고리 변경
- 검색 반경 자동 확장
- 추천 점수와 개인화
- 세션 만료
- 인프라와 배포

## 4. API

```http
POST /api/recommendations/sessions/{sessionId}/next
```

정상 응답은 `200 OK`이며 응답 형식은 세션 생성 응답과 같다.

## 5. 처리 흐름

1. Service가 세션 ID로 `RecommendationSession`을 조회한다.
2. 세션이 이전에 추천한 식당 ID를 확인한다.
3. 아직 추천하지 않은 운영 중 식당 하나를 반환한다.
4. 새로 추천한 식당 ID를 세션에 기록한다.
5. 변경된 세션을 Repository에 저장한다.

## 6. 비즈니스 규칙

- 같은 세션에서는 같은 식당을 두 번 추천하지 않는다.
- 존재하지 않는 세션은 `RECOMMENDATION_SESSION_NOT_FOUND`로 처리한다.
- 모든 후보를 사용하면 세션 상태를 `EXHAUSTED`로 바꾸고 `RECOMMENDATION_EXHAUSTED`를 반환한다.
- 선택을 완료한 세션에서는 다음 추천을 받을 수 없다.
- 같은 세션의 상태 변경 요청은 한 번에 하나씩 처리한다.

## 7. 테스트 시나리오

- 다음 추천은 첫 추천과 다른 식당을 반환한다.
- 존재하지 않는 세션은 404를 반환한다.
- 후보를 모두 사용하면 409를 반환한다.
- 도메인에서 비활성 식당과 중복 식당을 제외한다.
- 같은 세션에 동시에 요청해도 서로 다른 식당을 추천한다.

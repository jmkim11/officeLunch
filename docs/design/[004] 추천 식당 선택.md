# [004] 추천 식당 선택

## 1. 목적

사용자가 실제로 추천받은 식당 하나를 최종 선택하고 추천 세션을 종료할 수 있게 한다.

## 2. 구현 범위

- 추천 세션과 식당 ID 입력 검증
- 추천 이력에 포함된 식당인지 확인
- 선택 식당 ID 기록
- 세션 상태를 `SELECTED`로 변경
- 선택 결과 응답
- 선택 후 재추천과 중복 선택 차단

## 3. 제외 범위

- 선택 이력 통계
- 사용자별 선택 이력
- 즐겨찾기와 리뷰
- 외부 지도 길찾기 URL 생성
- 인프라와 배포

## 4. API

```http
POST /api/recommendations/sessions/{sessionId}/selection
Content-Type: application/json
```

```json
{
  "restaurantId": 1
}
```

정상 응답은 `200 OK`다. 응답의 `status`는 `SELECTED`이고 `restaurant`에는 선택한 식당 정보가 들어간다.

## 5. 처리 흐름

1. Controller가 식당 ID를 검증한다.
2. Service가 세션 ID로 기존 세션을 조회한다.
3. `RecommendationSession`이 해당 식당을 실제 추천했는지 확인한다.
4. 선택 식당 ID와 `SELECTED` 상태를 기록한다.
5. 변경된 세션을 Repository에 저장한다.
6. 선택한 식당 정보를 응답한다.

## 6. 비즈니스 규칙

- 식당 ID는 1 이상의 값이어야 한다.
- 현재 세션에서 추천받은 식당만 선택할 수 있다.
- 선택이 끝난 세션에서는 다른 식당을 다시 선택할 수 없다.
- 선택이 끝난 세션에서는 다음 추천을 받을 수 없다.
- 후보가 소진된 뒤에도 이미 추천받은 식당은 선택할 수 있다.

## 7. 테스트 시나리오

- 추천받은 식당 선택 시 상태와 선택 ID를 저장한다.
- 추천받지 않은 식당은 선택할 수 없다.
- 선택 완료된 세션은 다시 선택할 수 없다.
- 선택 완료된 세션은 다음 추천을 받을 수 없다.
- 식당 ID가 0이면 표준 400 오류를 반환한다.

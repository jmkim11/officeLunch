


## OFFICE LUNCH 전체 시스템 흐름
- 1. 사용자가 회사 이름을 입력한다.
- 2. 프론트엔드가 회사 검색 API 를 호출한다.
- 3. 백엔드가 외부 장소 검색 API 를 호출한다.
- 4. 검색 결과를 회사 정보 형태로 변환한다.
- 5. 사용자가 회사를 선택한다.
- 6. 백엔드가 회사 주변 식당을 조회한다.
- 7. 조회가 완료되었다면 프론트엔드에 완료 상태를 반환한다.
- 8. 프론트엔드가 사용자에게 추천 시작하기 버튼을 표시한다.
- 9. 사용자가 추천 시작하기 버튼을 누르면, 추천 세션이 시작된다.
- 10. 사용자가 음식 카테고리를 선택한다.
- 11. 추천하지 않은 식당 중 하나를 반환한다.
- 12. 사용자는 다시 추천하기를 선택하거나 식당을 선택한다.
- 13. 선택 결과를 저장하고 세션을 완료한다.

### 회사 검색
- 사용자 행동: 사용자가 회사 이름을 입력하고 검색한다.
- 프론트 요청: 회사 이름 검색어를 백엔드에 전달한다.
- 백엔드 처리: 외부 장소 검색 API 를 호출한다. 외부 응답을 회사 검색 결과 형식으로 변환한다.
- 반환 결과: 회사 이름, 도로명 주소, 위도, 경도

### 주변 식당 준비
- 사용자 행동: 검색 결과에서 회사를 선택한다.
- 프론트 요청: 선택한 회사의 위치 정보를 전달한다.
- 백엔드 처리: 회사 주변 식당을 외부 API에서 조회하고 후보 목록을 준비한다.
- 반환 결과: 식당 준비 완료 여부와 조회 결과 식별자

### 추천 시작
- 사용자 행동: 추천 시작하기 버튼을 누른다.
- 프론트 요청: 준비된 식당 조회 결과의 식별자를 전달한다.
- 백엔드 처리: 준비된 후보 목록으로 추천 세션을 생성한다.
- 반환 결과: 추천 세션 ID 와 세션 상태

### 식당 추천
- 사용자 행동: 음식 카테고리를 선택한다.
- 프론트 요청: 추천 세션 ID와 음식 카테고리를 전달한다.
- 백엔드 처리: 해당 카테고리의 식당 중 아직 추천하지 않은 식당 하나를 선택한다.
- 반환 결과: 추천 식당 한 곳과 세션 상태

### 다시 추천
- 사용자 행동: 다시 추천하기 버튼을 누른다.
- 프론트 요청: 추천 세션 ID를 전달한다.
- 백엔드 처리: 기존 추천 식당을 제외하고 다음 식당을 선택한다.
- 반환 결과: 새로운 추천 식당 한 곳과 세션 상태

### 추천 완료
- 사용자 행동: 추천받은 식당을 선택한다.
- 프론트 요청: 추천 세션 ID와 식당 ID를 전달한다.
- 백엔드 처리: 선택 가능 여부를 확인하고 식당 ID를 저장한 뒤 세션을 완료한다.
- 반환 결과: 선택된 식당 정보와 완료 상태

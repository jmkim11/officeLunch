# Office Lunch MVP 요구사항

## 1. 문서 목적

Office Lunch의 첫 번째 출시 범위를 정의한다.

개발 중 새로운 아이디어가 생기더라도 이 문서에 정의된 핵심 기능을
우선 완성하여 과도한 범위 확장을 방지한다.

## 2. 해결하려는 문제

직장인은 점심시간마다 회사 주변의 식당을 검색하고,
동료들과 메뉴를 결정하는 과정을 반복한다.

Office Lunch는 회사 주변 식당 중 조건에 맞는 한 곳을 추천하여
사용자의 점심 메뉴 결정을 돕는다.

## 3. 목표 사용자

- 회사 주변에서 점심을 먹는 직장인
- 점심 메뉴를 빠르게 결정하고 싶은 사용자
- 주변 식당을 잘 모르는 사용자

## 4. MVP 목표

사용자가 회사를 선택한 후 회사 주변 식당을 추천받고,
최종 선택한 식당을 확인할 수 있도록 한다.

## 5. 핵심 사용자 흐름

1. 사용자가 메인 화면에 접속한다.
2. `회사 정보 입력하기` 버튼을 누른다.
3. 회사명 또는 주소를 검색한다.
4. 검색 결과에서 회사를 선택한다.
5. 선택한 회사 주변의 식당 후보를 조회한다.
6. 음식 카테고리를 선택한다.
7. 조건에 맞는 식당 한 곳을 추천받는다.
8. 추천을 수락하거나 다시 추천받는다.
9. 최종 선택한 식당 정보를 확인한다.
10. 외부 지도 서비스를 통해 길찾기를 실행한다.

## 6. MVP 기능 요구사항

### 6.1 회사 검색

- 사용자는 회사명 또는 주소로 회사를 검색할 수 있어야 한다.
- 시스템은 검색 결과로 장소명과 도로명 주소를 제공해야 한다.
- 사용자는 검색 결과 중 하나를 회사로 선택할 수 있어야 한다.
- 선택된 회사의 위치 정보를 추천 과정에서 사용할 수 있어야 한다.

### 6.2 주변 식당 조회

- 시스템은 선택된 회사 위치를 기준으로 주변 식당을 조회해야 한다.
- 검색 반경의 초기값은 500m로 한다.
- 식당의 이름, 카테고리, 주소, 거리 및 위치 정보를 관리해야 한다.
- 조회 결과가 없으면 사용자에게 안내해야 한다.

### 6.3 식당 추천

- 사용자는 음식 카테고리를 선택할 수 있어야 한다.
- 시스템은 조건에 맞는 식당 후보 중 한 곳을 추천해야 한다.
- 사용자는 현재 추천을 거절하고 다시 추천받을 수 있어야 한다.
- 추천 후보가 없으면 사용자에게 안내해야 한다.

### 6.4 식당 선택

- 사용자는 추천된 식당을 최종 선택할 수 있어야 한다.
- 시스템은 사용자가 선택한 식당을 기록해야 한다.
- 선택 결과 화면에서 식당 정보를 확인할 수 있어야 한다.

### 6.5 길찾기

- 사용자는 선택 결과 화면에서 길찾기를 실행할 수 있어야 한다.
- 길찾기는 외부 지도 서비스의 URL을 이용한다.
- MVP에서는 자체 지도와 경로 안내 기능을 구현하지 않는다.

## 7. 비기능 요구사항

### 7.1 성능

- 일반적인 API 요청은 목표 응답 시간 이내에 처리되어야 한다.
- 외부 장소 API 호출이 지연될 경우 무제한으로 대기하지 않아야 한다.

> 구체적인 응답 시간과 타임아웃 값은 기능별 Design Doc에서 결정한다.

### 7.2 안정성

- 외부 장소 API 장애가 내부 서버 전체의 장애로 이어지지 않아야 한다.
- 잘못된 요청에 대해 일관된 오류 응답을 제공해야 한다.
- 추천하거나 선택할 수 없는 식당을 처리해야 한다.

### 7.3 데이터 정합성

- 존재하지 않는 회사나 식당을 선택할 수 없어야 한다.
- 한 번의 선택 요청으로 선택 기록이 중복 저장되지 않아야 한다.
- 추천 결과와 최종 선택 기록을 구분해야 한다.

### 7.4 테스트

- 핵심 비즈니스 로직에 단위 테스트를 작성해야 한다.
- 주요 API의 정상 및 예외 흐름을 검증해야 한다.
- 외부 API에 의존하지 않고 핵심 테스트를 실행할 수 있어야 한다.

## 8. MVP 제외 범위

다음 기능은 첫 번째 출시 범위에서 제외한다.

- 회원가입 및 로그인
- 사용자별 장기 개인화 추천
- 여러 사용자의 실시간 투표
- 리뷰 작성
- 즐겨찾기
- 결제 및 예약
- 관리자 화면
- 자체 지도 및 경로 탐색
- AI 기반 추천
- 모바일 애플리케이션

제외된 기능은 MVP가 완료된 후 사용자 반응과 필요성을 검토하여 추가한다.

## 9. 완료 조건

다음 조건을 모두 충족하면 MVP가 완료된 것으로 판단한다.

- 사용자가 회사를 검색하고 선택할 수 있다.
- 회사 주변의 식당 후보를 조회할 수 있다.
- 카테고리에 맞는 식당을 추천받을 수 있다.
- 사용자가 식당을 최종 선택할 수 있다.
- 선택 결과를 확인하고 외부 지도 길찾기로 이동할 수 있다.
- 핵심 정상 흐름과 예외 흐름에 대한 테스트가 통과한다.
- 실제 배포 환경에서 핵심 사용자 흐름을 수행할 수 있다.

## 10. 보류 사항

다음 내용은 기능별 Design Doc에서 결정한다.

- 회사 정보의 저장 방식

- 추천 및 선택 이력의 데이터 모델
- 외부 지도 연동 방식
- API 오류 응답 형식
